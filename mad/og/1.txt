File 2: MainActivity.java (validation included)
java
package com.example.loginvalidation;

import androidx.appcompat.app.AppCompatActivity;
import android.os.Bundle;
import android.widget.EditText;
import android.widget.Button;
import android.widget.Toast;

public class MainActivity extends AppCompatActivity {
    EditText etUsername, etPassword;
    Button btnLogin;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        etUsername = findViewById(R.id.etUsername);
        etPassword = findViewById(R.id.etPassword);
        btnLogin = findViewById(R.id.btnLogin);

        btnLogin.setOnClickListener(v -> {
            boolean valid = true;
            if (etUsername.getText().toString().trim().isEmpty()) {
                etUsername.setError("Username required");
                valid = false;
            }
            if (etPassword.getText().toString().trim().isEmpty()) {
                etPassword.setError("Password required");
                valid = false;
            } else if (etPassword.getText().toString().length() < 8) {
                etPassword.setError("Password must be at least 8 characters");
                valid = false;
            }

            if (valid) {
                Toast.makeText(this, "Login successful", Toast.LENGTH_SHORT).show();
                // Optionally go to next screen
            }
        });
    }
}

Essential Viva/Exam Questions & Answers
1. What is form validation?
It is the process of verifying if user inputs are correct, complete, and in expected format before processing them further. For example: not allowing a blank username or a short password.

2. Where do we perform login form validation in Android?
Typically inside the Activity/Fragment in the button click listener. We check each EditText for rules using methods like getText().toString() and setError().

3. What happens if validation fails?
Show error to user (using setError, Toast, or changing field color). Prevent moving forward until all fields are valid.

4. Explain how you'd validate an email address.
Use Patterns.EMAIL_ADDRESS.matcher(email).matches() to check format. In XML/Java, set error if mismatch.

5. How do you show errors to users in EditText?
Use setError("Message") on the specific EditText view. This highlights the field and pops a message below it.

6. What are common fields validated in login/registration forms?
*- Username/Email (not blank, right format)

Password (length, sometimes characters number/type)

Optional: Phone number, name, date of birth, gender (using radio buttons), etc.

7. What is inputType in XML?
It restricts the keyboard and input for that field: e.g., inputType="textPassword" shows a password keyboard and masks characters.

8. How do you clear all errors if the user corrects their input?
Error is cleared automatically once input is valid, or can be cleared by logic (e.g. after successful submit, use setError(null)).

9. Why is form validation important?
*- Prevents incomplete/incorrect data entry

Improves security (e.g. don't send blank passwords)

Better user experience (immediate feedback)

Reduces backend errors/data inconsistencies*
